# Vision Preview Rail Plan

## Status

Completed.

## Goal

Keep the `/vision` preview rail compact and contextual so it supports drafts, review-ready states, and resolved details without becoming a second form system.

## Scope

- Included: preview rail layout, density, and contextual visibility rules.
- Excluded: terminal redesign and worklist rendering.

## Steps

1. Keep the rail focused on draft and review states.
2. Remove extra form-like noise where the terminal already carries the interaction.
3. Make the preview easier to scan and less visually dominant than the terminal.

## Validation

- `npm run type-check`
- `npm run build`

## Completion

- The preview rail is compact and contextual instead of card-heavy.
- Structured preview fields use a row-based layout aligned with the terminal grammar.
