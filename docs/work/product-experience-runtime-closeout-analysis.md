# Runtime closeout analysis

The closeout must combine static contract evidence with authenticated browser traces. It must cover shell title deduplication, the single global Vision entry, account menu and profile media recovery, all explicit discovery entries, multiple-business switching, favorite-to-book handoff, and commute consent/no-leakage behavior. A pending runtime scenario remains open even when source tests and builds pass. The master verifier may close only after every child plan is independently verified and `make audit-runtime-acceptance` reports no pending scenarios.
