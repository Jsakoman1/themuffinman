# Shell IA deduplication analysis

The authenticated shell currently owns navigation, account text, logout, and a Vision handoff, while several surface views repeat module labels, titles, and actions. The implementation must establish one route metadata contract for canonical title, hierarchy label, contextual Vision prompt, and page actions. Deleting a repeated control is safe only after its destination is mapped to the shell, a page action, or a detail surface. Responsive and keyboard runtime evidence is required because shell compression can hide the only account or Vision entry.
